**Issues — actionable**

1. **Scope too big for v1**. 21 units, 6 phases. Unit 8 alone freezes full LSP-framed JSON-RPC surface (init/initialized, capabilities, declared_outputs, cancel, shutdown, progress, timeouts, cookie, `_meta`) + 2 reference adapters (Go + Python) + conformance harness. 2. **Protocol freeze before third-party use = premature lock-in**. Rule-of-Three violated. `capabilities{}` is additive so minor bumps safe, but `declared_outputs` + LSP-extension error codes + `_meta` added with zero field usage. 3. **Tool-owned-file merges under-specified — biggest data-loss surface**. Decision #25 names `write_tool_owned` with json-pointer/toml-path/markdown-section locators, but:

   - Markdown sections: recovery when user deletes one `<!-- aienvs:end -->`? Nests a begin inside a begin? Not specified.
   Recommend: lift tool-owned-file handling into its own unit (12a) with explicit round-trip fixtures for each format. Currently spread across units 9/10/11 with scenarios saying "merge preserves user content" but no mechanism.

4. **`AGENTS.md` concurrent writes by cursor + codex adapters unlocked**. Both merge into workspace-root `AGENTS.md` in parallel staging under atomic mode. Unit 12 lock is per-target. No per-external-file lock. Section markers keyed by `<id>` solve *logical* collisions but not *write-race* collisions. Add: orthogonal per-tool-owned-file flock (path → lock) in unit 12.
