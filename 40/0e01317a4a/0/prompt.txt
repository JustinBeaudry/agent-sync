Note the current gate ignores `--target`: passing only `--target` on a TTY still launches the wizard; passing any source flag skips it. **3. Manifest schema and writing**

- Schema: `internal/manifest/schema.go` — `Manifest{Version, Canonical CanonicalSource, Targets []string, Scope string, Cache, Adapters []AdapterDecl, TrustedSHA, Trust, Compose}`. A discovery feature that snapshots targets at init changes that empty-is-valid ergonomic story but not the schema.
- `Scope` field already exists in the schema (`user | project | global`, "informs the adapters where the emitted files live"), separate from the runtime scope passed to `prepareScope`.
- Writing: `internal/manifest/write.go` — `WriteFile(path, content)` opens the **parent dir** via `fsroot.OpenWorkspaceRoot` and uses `root.StagedWrite` (atomic, crash-safe, 0o644). `WriteResolvedSHA`/`WriteTrustedSHA` do AST-preserving in-place key updates and require the key to pre-exist (`ErrKeyMissing`) — relevant if discovery ever wants to *update* `targets:` in an existing manifest rather than write a fresh one.

**6. fsroot constraints for filesystem probing**

- `internal/fsroot` (`root.go`, `containment.go`, `safewrite.go`) is the **write** enforcement point. Decision axes: source flags (none / `--source` / `--local-path` / `--local-dir`) × targets (none / `--target…`) × mode (interactive TTY / non-interactive / accessible-TTY) × discovery outcome (some / zero / stat-error). **Non-interactive (or piped stdin):**

## (b) Edge cases / state transitions

- **`.claude` is a file, not a dir** → plan says "os.Stat on dir names"; must add an `IsDir()` check or a file named `.pi` fires discovery.
- **Symlinked footprint dir** → `os.Stat` follows symlinks; a dangling symlink is not discovered. Fine, but state it.
- **Stat error other than NotExist** (permission-denied dir) → skip-silently vs fatal? Existing init precedent treats non-NotExist stat as fatal (`cmd_init.go:104-107`); for a 5-dir probe, fatal on one unreadable dir is harsh. Decide and document.
- **Case-insensitive APFS/NTFS**: `.CLAUDE` will satisfy `os.Stat(".claude")` on the default macOS filesystem. Accept as minor; don't try to fight it.
- **`.agents` vs `.agent`**: the source default and antigravity's footprint differ by one character. Users will typo `--local-dir .agent`; error copy and docs should keep these visually distinct.
- **Ordering**: discovered targets must be written sorted (wizard already sorts, `wizard/init.go:79`); a map-driven probe would otherwise produce nondeterministic `targets:` order — noisy diffs, flaky tests.
- **Duplicate `--target claude --target claude`** → duplicate manifest entries. Pre-existing; spec says explicit overrides discovery with no merge, so no new dedup path needed, but worth a normalizing pass while touching this.
- **Wizard with zero targets selected** → today Enter proceeds to confirm with empty `Targets` → valid empty-targets manifest. Fine; keep, but a confirm-screen "targets: (none)" render is needed (currently renders an empty join).
- **Existing-manifest check ordering**: today the user completes the whole wizard before hitting "already exists" (`cmd_init.go:101`). Adding discovery makes the pre-write work heavier; move the stat before wizard/discovery while in here.

## (c) Ambiguities the plan must resolve (prioritized)

**Must-resolve**

1. **Item 1 vs item 4 contradiction + breaking change.** In a fresh empty repo, bare `agent-sync init --non-interactive` has no source flags *and* zero footprints — item 1 says it succeeds, item 4 says it fails. Also, `init --non-interactive --source URL` with no `--target` succeeds *today* (empty `targets:` is explicitly spec-valid — `manifest/schema.go:13` "a valid state for a workspace that is not yet configured"), and item 4 would make it fail. **Recommended default:** write the empty-targets manifest with a stderr hint ("no targets discovered; add with `--target` or edit targets: in .agent-sync.yaml") rather than `MissingFlagError`. That preserves item 1, avoids the regression, and the existing zero-emit hint (#37) plus `engine.ErrAdapterNotFound` already backstop sync. If fail-fast is truly wanted, scope it to *bare* init only and say so.
2. **Pin flags × defaulted source.** `--ref`/`--commit`/`--floating` with no source flag must produce a purpose-built error ("--ref requires --source or --local-path; the default .agents source is unpinned"), not the generic LocalDir validation error from `initconfig.go:63-67`. Decide: error (recommended) vs suppress-the-default.
3. **Wizard LocalDir mechanics.** The wizard has *no* LocalDir path: `looksLikeLocalPath` would classify a typed `.agents` as **LocalPath**, sending it into `git.ResolveLocalRef` → failure. "Source screen gains a `.agents` default" needs concrete design: (a) empty-Enter accepts LocalDir=`.agents` (today empty Enter is rejected, `wizard/init.go:118-120`); (b) LocalDir selection must **skip phaseRef** (ref+local_dir is invalid); (c) `sourceDisplay()` and confirm screen must render local_dir; (d) how does a user type a *custom* local-dir vs a local git path — probably a source-type choice, not string heuristics. Also: discovery is fs I/O — do it in `cmd_init.go` and pass results into `wizard.Run` (new parameter), keeping the wizard I/O-free per KTD-4.
4. **`.agents` missing at sync.** Manifest is valid, but first `sync` hard-fails with `worktree.ErrSourceMissing` (`worktree/reader.go:43`). Bare init succeeding into a guaranteed sync failure is a bad handoff. **Recommend:** create `.agents/` at init when defaulted (via `fsroot.OpenWorkspaceRoot` — the CLAUDE.md invariant; reads may os.Stat, writes may not) — an *empty* existing dir degrades to the soft zero-emit hint instead of a hard error. Alternative: print "create .agents/ before running sync". Pick one.
5. **Does `--target` alone on a TTY still launch the wizard?** (Flow 11.) Today yes. If the source default applies, the invocation becomes fully specified and should skip the wizard — but that's a silent behavior change for TTY users. Decide and test explicitly; the wizard-gate condition at `cmd_init.go:66` is the exact line in question.

**Important**

6. **Probe base + bad `--dir` error ordering.** Probe `destDir`; stat `destDir` itself first so a nonexistent `--dir` fails with "no such directory", not "pass --target".
7. **Home-dir semantics.** Either (a) declare discovery project-scope-only and document that `init --dir ~` may under-discover (antigravity's user footprint is `.gemini`), or (b) add per-scope footprints. Given init has no `--user` flag, (a) is the lean answer — but write it down.
8. **Zero-match wizard preselection.** Recommend: discovered set pre-selected when non-empty; **all** pre-selected when zero discovered (preserves today's greenfield UX and makes zero-match non-punishing). "None preselected" + Enter yields an empty-targets manifest one keystroke away — hostile default.
9. **Probe list source of truth.** Derive from `bundledAdapters()[i].Manifest.ReservedPrefix` (already validated, present on all five), not a hardcoded map — new bundled adapters then get discovery for free. Decide explicitly that PATH adapters (`internal/adapter/discover.go`, which carry `reserved_prefix` but require subprocess execution to read) do **not** participate in v1, and say why (no subprocess exec at init).
10. **Accessible mode.** Pin flow 13's behavior in tests: default + discovery apply, silent success is acceptable, zero-match follows whatever Gap 1 decides.
11. **Announce the magic.** Success output must name what was defaulted/discovered: `wrote .agent-sync.yaml (source: .agents [default]; targets: claude, cursor [discovered])`. A defaulting feature that prints only `wrote .agent-sync.yaml` hides misdiscovery until sync.

**Nice-to-have**

12. Warn (don't fail) at init when an explicit `--target` name is neither bundled nor `agent-sync-adapter-&lt;name&gt;` on PATH — must stay non-fatal (adapters may be installed later / on other machines; `ErrAdapterNotFound` at sync remains the enforcement).
13. When explicit `--target` overrides discovery, note skipped discoveries: "also detected .cursor (not enabled)".
14. Staleness: targets are snapshotted; a later-added `.cursor` never joins. A future sync-side hint ("footprint .cursor present but cursor not in targets") or `agent-sync update` rediscovery is out of scope — log it as such.
15. Move the existing-manifest stat before wizard/discovery.
