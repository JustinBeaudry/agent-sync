<approach>
**Execution order**: Units 1 → 2 → 3, then 4 → 5 in parallel-conceptual but actually serial since both adapterkit units land in `pkg/adapterkit/` and 5 depends on 4. Recommended sequence: 1, 2, 3, 4, 5 (strictly serial for clean commits).

**Unit 1 approach:**
- Package doc explains the harness's role: spec freeze enforcement, conformance for third-party adapters.
- `Case` struct: `Name string`, `Description string`, `IR json.RawMessage`, `Manifest CaseManifest` (declared_outputs + capabilities), `Expect Expected` (tagged union: `Ops []OpRecord`, `Error string` sentinel name, or `Skip string` reason).
- `Expected` discriminator field: `kind: "ops" | "error" | "skip"`.
- `LoadCorpus()` reads every `corpus/*.json` from embed.FS, parses into `[]Case`, sorts deterministically by Name. Returns `(cases, error)`.
- `OpRecord` here mirrors `contract.OpRecord` (you may import `internal/adapter/contract` for this — conformance lives at `internal/adapter/conformance/`, not in `pkg/`, so internal imports are fine).
- Validate at load: empty `Name` rejected, both `Ops` and `Error` set in same Expect rejected, kind discriminator matches the populated field.

**Unit 2 approach:**
- Each fixture is one self-contained JSON document with shape:
  ```
  {
    "name": "happy-rule",
    "description": "Adapter handles a single rule kind, emits one mkdir + one write_file under .echo/",
    "ir": {"nodes": [{"id": "no-fri", "kind": "rule", ...}]},
    "manifest": {
      "declared_outputs": [{"path": ".echo/rules", "mode": "owned-subdir"}],
      "capabilities": {"concept_kinds": {"rule": "supported"}}
    },
    "expect": {
      "kind": "ops",
      "ops": [
        {"op": "mkdir", "path": ".echo/rules"},
        {"op": "write_file", "path": ".echo/rules/no-fri.md"}
      ]
    }
  }
  ```
- The `name` field MUST equal the filename without `.json`.
- Adversarial fixtures: `expect.kind = "error"` with the expected sentinel name (one of: `ErrAdapterProtocolOrderViolation`, `ErrAdapterCookieMismatch`, `ErrAdapterProtocolMismatch`, `ErrAdapterUndeclaredOutput`, `ErrAdapterCapabilityLied`, `ErrAdapterTimeout`, plus a generic non-zero exit class for `error-abnormal-exit`).
- Add a `corpus_completeness_test.go` (or extend `corpus_test.go`) — for every kind in `ir.AllKinds()`, assert at least one corpus case has that kind in its IR.

**Unit 3 approach:**
- `RunOptions` struct: `Cases []Case` (default: `LoadCorpus()`), `Timeouts adapter.SubprocessTimeouts` (default: runtime defaults), `Verbose bool`.
- `Report` struct: `Cases []CaseResult`, `Summary{Total, Passed, Failed, Skipped int}`.
- `CaseResult`: `Name string`, `Status string` (`pass`/`fail`/`skip`), `Reason string`, `ActualOps []OpRecord`.
- For each case:
  1. Build an `adapter.AdapterManifest` from `case.Manifest`: `Command: []string{binary}`, `ContractVersion: ContractVersionV1`, `Name: "conformance-target"`.
  2. Construct `adapter.Adapter{Source: SourcePATH, Manifest: ...}`.
  3. Spawn session via `adapter.NewSession(ctx, opts)`.
  4. Drive `Initialize` → `Initialized` → `Emit(case.IR)` → `Shutdown`.
  5. If `case.Expect.Kind == "ops"`: compare emitted ops against expected (order-insensitive set match unless `expect.strict_order: true`).
  6. If `case.Expect.Kind == "error"`: check returned error matches the expected sentinel via `errors.Is` against the named sentinel from `internal/adapter/errors.go`.
  7. If `case.Expect.Kind == "skip"`: mark skip with reason. Skips do not count as failures in the summary.
- `assertions.go`: `MatchOps(expected, actual []OpRecord) (ok bool, missing, extra []OpRecord)`; `MatchError(expectedSentinel string, err error) bool` (sentinel name → reflect lookup or hand-coded switch).
- Tests: run harness against `internal/adapter/testdata/echo/main.go` (built via `buildTestdataBinary`-equivalent) — most cases will skip (echo only handles `rule`), `happy-rule` should pass. Run against `internal/adapter/testdata/crashy/main.go` — every case fails with abnormal-exit. Test ctx cancellation, empty corpus, op-diff order semantics.

**Unit 4 approach:**
- **Mirror types** in `pkg/adapterkit/types.go` with identical JSON tags and field names to `internal/adapter/contract`. Different Go types — no aliases. Schema parity test catches drift.
- Public types (mirror from contract): `InitializeParams`, `InitializeResult`, `Capabilities`, `DeclaredOutput`, `EmitParams`, `EmitResult`, `OpRecord`, `OpKind` (string type) + constants `OpKindWriteFile`/`OpKindWriteToolOwned`/`OpKindMkdir`/`OpKindDelete`/`OpKindWarning`, `OutputMode` + constants, `CapabilityLevel` + constants, `Error`, `ErrorClass` + constants, method-name constants (`MethodInitialize`, `MethodInitialized`, `MethodEmit`, `MethodShutdown`).
- **Server** (final shape decided during execution; the reference echo in next batch is the forcing function):
  ```
  type Server struct { ... }
  type InitializeFunc func(ctx context.Context, params InitializeParams) (InitializeResult, error)
  type EmitFunc func(ctx context.Context, params EmitParams) (EmitResult, error)
  type ShutdownFunc func(ctx context.Context) error
  func NewServer(name, version string) *Server
  func (s *Server) OnInitialize(fn InitializeFunc)
  func (s *Server) OnEmit(fn EmitFunc)
  func (s *Server) OnShutdown(fn ShutdownFunc)
  func (s *Server) Run(ctx context.Context) error  // reads stdin, writes stdout
  ```
- The Server handles: magic-cookie env read (`AIENVS_ADAPTER_COOKIE`; exits 7 if missing), version validation (refuses if client speaks anything other than `aienvs/v1`), writes "started" to stderr (matches existing testdata echo convention), recovers handler panics and reports them as `adapter-panic` errors.
- **Schema parity test** (`schema_parity_test.go`): for each public adapterkit type with a corresponding JSON schema, marshal a sample value to JSON and validate it against the schema. Use the new `contract.LoadSchema(name)` accessor (added as the small public addition to `internal/adapter/contract/schema.go`). Failure means adapterkit drifted from contract.
- The `LoadSchema` addition to contract: a 5-line function that wraps `SchemaFS.ReadFile("schema/" + name + ".json")` with proper error wrapping. This is the only modification to `internal/adapter/contract/` in this batch.

**Unit 5 approach:**
- **CapabilitiesBuilder** fluent API:
  ```
  caps := adapterkit.NewCapabilities().
    Supports("rule").
    Partial("skill", "agent assets not yet handled").
    Unsupported("mcp-server-entry").
    WithWriteToolOwned(true).
    Build()
  ```
- **Testing helpers** (exported, in `pkg/adapterkit/testing.go` — note: NOT `_test.go`, must be a regular file with exported APIs since third-party authors use them):
  - `ScriptedResponder` — exported version of `internal/adapter/runtime_test.go::scriptedAdapter`. Lets adapter authors write unit tests against synthetic clients.
  - `SynthesizeInitResult(name, version string, caps Capabilities, outputs []DeclaredOutput) []byte` — returns wire bytes of a valid `InitializeResult`.
  - `RunInprocServer(t, server) (client *Client, cleanup func())` — wires Server to in-memory `io.Pipe` pair so tests drive it without real subprocess.
  - `AssertProtocolShutdownAcked(t, server)` — verifies server called the equivalent of `MarkProtocolShutdownAcked` in its shutdown flow.
- **Example_* functions** in `example_test.go`: one per public entry point (`Example_NewServer`, `Example_CapabilitiesBuilder`, `Example_RunInprocServer`). Per the spec-impl-drift learning, these lock prose docs to compiling code.
